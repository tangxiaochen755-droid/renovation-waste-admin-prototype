# 装修垃圾便民服务后台管理原型

## 打开方式

直接双击 `index.html`。英文备用入口为 `admin-prototype.html`。

本项目没有构建步骤、没有外部 CDN、无需安装 Node.js，解压后可离线演示。

## 当前范围

第一轮最小可运行原型已覆盖：

- 3种后台角色切换：系统管理员、业务管理员、审核人员
- 角色化工作台
- 运送登记列表、筛选和详情
- 三张照片与三点定位审核
- 处置码列表
- 延期审批
- 核销与退回记录
- 作废、异常标记等模拟交互
- 堆放点新增、编辑与启停
- 内容新建、编辑、发布与下线
- 小程序用户详情、封禁与解封
- 三角色权限矩阵与保存确认
- 处置码、延期、照片定位及区域围栏配置
- 后台关键操作日志

原型已内置一套初版照片、定位、处置码、延期、核销和异常处理规则，具体规则见 `docs/PRD.md`。

## 文件结构

- `index.html`：中文主入口
- `admin-prototype.html`：英文文件名备用入口
- `styles/`：Design Tokens 映射和界面样式
- `scripts/mock-data.js`：可直接离线加载的模拟数据
- `scripts/app.js`：导航、页面渲染和交互逻辑
- `tokens/design-tokens.json`：设计变量源文件
- `docs/PRD.md`：首轮原型需求和业务规则

## 演示建议

1. 从右上角切换三种角色，查看工作台和菜单差异。
2. 进入“运送登记”，打开任意记录详情。
3. 进入“照片定位审核”，切换审核队列并模拟标记异常。
4. 进入“延期审批”，完成一次审批演示。
5. 进入“核销记录”，查看已核销与已退回详情。
6. 使用业务管理员演示堆放点、内容和用户管理。
7. 切换系统管理员演示角色权限、业务规则和操作日志。

## 当前限制

- 所有数据仅保存在页面内存中，刷新后恢复初始模拟数据。
- 不连接小程序、数据库、地图或真实接口。
- 当前规则为第一版业务基线，后续根据客户反馈直接迭代。
