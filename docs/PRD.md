# 装修垃圾便民服务后台管理原型 PRD

## 文档定位

本文档用于第一轮客户核对版原型，不是最终研发需求。小程序已经存在，本项目仅设计与其对应的 PC 管理后台。

## 首轮目标

用可离线打开的静态原型验证后台信息架构、三类角色权限和核心业务闭环。优先保证流程能演示，不追求真实接口和完整配置能力。

## 角色

### 系统管理员

- 查看全部业务数据。
- 管理后台账号、角色权限、全局规则、区域和操作日志。
- 可以进入全部业务页面。

### 业务管理员

- 管理登记、处置码、延期、核销记录、堆放点、内容和小程序用户。
- 可执行延期审批、作废、用户处置等业务动作。
- 不管理后台系统级权限和审计配置。

### 审核人员

- 查看授权范围内的运送登记、处置码和核销结果。
- 审核照片与定位、标记异常并填写依据。
- 不允许修改系统规则、发布内容、封禁用户或管理账号。

小程序核销员不属于 PC 后台角色，继续在现有小程序中扫码核销。

## P0页面

### 角色化工作台

- 根据角色显示不同的指标、待办和风险。
- 支持切换角色进行客户演示。
- 工作台数据均能跳转至对应业务页面。

### 运送登记列表

- 字段：处置码、登记编号、登记人、手机号、垃圾来源、提交时间、有效期、照片状态、定位状态、业务状态。
- 支持关键词、状态、日期筛选。
- 支持查看详情、模拟导出和符合权限时的作废操作。
- 作废必须填写原因。

### 运送登记详情

- 展示登记基本信息、三张照片、三处定位、处置码和业务轨迹。
- 可进入审核工作台。
- 已核销记录默认只读。

### 照片与定位审核

- 展示待审核队列、风险等级和三张照片。
- 展示三处模拟定位和距离异常提示。
- 支持标记正常或异常。
- 标记异常不自动封禁用户。

### 处置码管理

- 展示编号、关联登记、生成时间、有效期、状态和延期状态。
- 一次登记生成一个处置码，处置码默认有效期为2天，到期前6小时进入预警状态。
- 支持查看详情、延期审批和符合权限时的作废。

### 延期审批

- 展示申请人、原有效期、申请天数、原因和状态。
- 支持通过、拒绝和审批意见。
- 每个处置码仅可延期1次，每次固定延长1天；申请必须在处置码失效前提交，已失效后不可补申请。

### 核销记录

- 展示处置码、联单编号、核销员、扫码时间、完成时间、堆放点和结果。
- 后台首版只查看，不代替小程序核销。
- 核销退回后原处置码终止，用户需要重新完成运送登记并生成新的处置码。

## 后台管理页面

### 堆放点管理

- 展示名称、地址、区域、开放时间、联系人、电话和启停状态。
- 支持新增、编辑、启用和停用；变更后同步影响小程序服务须知展示。

### 内容管理

- 统一管理普法宣传、办理指南、备案指南、服务须知和温馨提示。
- 支持草稿、发布、下线和历史版本，发布后同步至小程序对应栏目。

### 用户管理

- 展示用户基本信息、累计登记、异常次数、最近使用和账号状态。
- 支持查看业务轨迹、封禁和解封；封禁原因及操作记录必须留痕。

### 角色权限

- 使用系统管理员、业务管理员、审核人员三类角色。
- 权限包含菜单访问、操作能力和数据范围；仅系统管理员可以修改权限。

### 业务规则配置

- 维护处置码有效期、到期预警、延期次数、拍摄时长、定位距离、GPS精度和区域围栏。
- 规则修改默认只影响保存后的新登记，并记录完整变更日志。

### 操作日志

- 记录操作人、角色、模块、动作、业务对象、时间和结果。
- 覆盖延期审批、异常审核、用户封禁、内容发布、权限与规则修改等关键操作。

## 初稿状态

- 运送登记：待核销、已核销、已退回、即将失效、已失效、作废。
- 处置码：待核销、已核销、已失效、作废。
- 延期申请：待审批、已通过、已拒绝。
- 审核：待审核、正常、异常。

## 初版业务规则

### 照片与定位

1. 空车、现场垃圾、装车三张照片必须通过小程序现场拍摄，不允许从相册选择。
2. 照片须按空车、现场垃圾、装车顺序完成，三张照片总时间跨度不得超过120分钟。
3. GPS定位精度须在100米以内；相邻拍摄点距离超过500米时标记为定位异常。
4. 垃圾产生地必须位于配置的服务区域围栏内，超出围栏标记为越界异常。
5. 照片与定位审核采用事后抽查，不阻塞现场正常核销；异常记录进入人工复核队列。
6. 审核人员标记异常时必须选择异常类型、风险等级并填写审核依据，异常标记不会自动封禁用户。

### 处置码与延期

1. 一次运送登记只生成一个处置码，处置码默认有效期为2天。
2. 到期前6小时标记为即将失效，并进入业务管理员预警队列。
3. 每个处置码仅可申请延期1次，审批通过后固定延长1天。
4. 延期申请必须在处置码失效前提交，已失效、已核销、已退回或已作废的处置码不可申请延期。
5. 业务管理员可以作废待核销、已失效或已退回的记录；已核销记录不可作废。
6. 作废必须填写原因且不可恢复，全部操作写入操作日志。

### 核销与退回

1. 核销由小程序端核销员现场扫码完成，PC后台不提供代核销。
2. 核销通过必须填写处置联单编号，完成后处置码不可再次使用。
3. 核销退回必须填写退回原因，原处置码随即终止。
4. 被退回的用户须重新完成运送登记并生成新处置码，不允许使用原码再次核销。

### 用户与统计

1. 用户封禁后禁止发起新登记，其尚未核销的处置码同步作废，历史记录继续保留用于审计。
2. 今日登记量按登记提交时间统计；今日核销量按核销完成时间统计。
3. 核销率按统计周期内核销完成量除以同期登记提交量计算。
4. 异常量包含照片异常、定位异常、核销退回和管理员作废记录，同一登记按一条异常记录计数。

## 成功标准

- 三种角色菜单和操作权限差异清晰。
- 登记、审核、处置码、延期和核销流程可演示。
- 关键异常操作要求填写原因或说明。
- 页面在 1366、1440、1920 宽度下不出现主体撑破。
- 125%浏览器缩放下核心表格可水平滚动且不遮挡操作。
- 全部资源本地化，解压后双击入口即可打开。
